
## 2024-05-18 - Interactive Ticker Cards Need Explicit Focus States
**Learning:** Custom interactive elements, such as the `.ecard` pattern used in the ticker track, are not natively keyboard accessible. Users relying on keyboard navigation cannot focus these elements, nor can they activate them since only click events were bound. Also, applying an outline directly to `.ecard` without an appropriate `outline-offset` or `z-index` causes visual clipping within the scrollable track container.
**Action:** Always add `tabindex="0"` to custom interactive elements. Implement a `keydown` listener that checks for `Enter` and `Space` keys to mirror the `click` functionality. When styling `:focus-visible` for items in a scrollable track, use `outline-offset` (often negative) or position/z-index adjustments to prevent the outline from being clipped by sibling elements or container bounds, ensuring the high-contrast outline is clearly visible. Use Nike's Volt color (`#ccff00`) for the outline.
