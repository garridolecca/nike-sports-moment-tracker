## 2024-05-24 - Interactive Ticker Card Accessibility
**Learning:** Custom interactive elements like the `.ecard` ticker items require explicit keyboard accessibility management. Simply adding `tabindex="0"` allows them to receive focus, but they also require dedicated `keydown` event listeners for `Enter` and `Space` keys to trigger their associated actions, as they don't natively trigger `click` events like buttons do. Furthermore, when adding focus outlines to elements within `overflow: hidden` containers (like the bottom ticker track), `outline-offset: -[size]` should be used so the focus indicator is inset and not clipped by the container bounds.
**Action:** When creating non-standard interactive UI elements that act like buttons, always bind `keydown` listeners for Enter/Space, manage focus order with `tabindex`, and test focus outline visibility within scrolling or clipping container boundaries.
