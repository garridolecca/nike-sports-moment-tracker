## 2024-05-24 - Explicit Focus Tracking for Custom UI Elements
**Learning:** In complex web applications like ArcGIS SceneViews, native browser focus behavior (e.g., sequential Tab navigation) can be unexpectedly trapped or misdirected. Therefore, when building custom interactive elements (like the `.ecard` ticker items), you must explicitly manage focus. Simply adding `tabindex="0"` is not enough; you must also explicitly handle keyboard interactions (like `keydown` on `Enter`/`Space`) to map them to the intended actions. Additionally, visual focus states (`:focus-visible`) should be styled distinctively (e.g., using a high-contrast theme color like Volt `#ccff00`) and may require inset spacing (e.g., `outline-offset: -2px`) to avoid being visually clipped by overflow containers like horizontal tickers.
**Action:** Always verify keyboard accessibility of custom UI components using explicit `.focus()` in tests rather than simulating sequential `Tab` presses. Ensure custom components have both `tabindex="0"` and explicit keydown handlers.
